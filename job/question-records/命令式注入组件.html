命令式注入组件（进度条组件） ​

常规做法 ​

  • 新建一个组件ImportExportProgress，然后在要使用的地方注册并挂载到页面上

bad: 组件并不是一直需要使用，这种方式会一直挂载到页面上，且反馈类的组件与业务无关，期望通过函数调用实现相同效果，且在不需要时不占用内存

命令式 ​

  • 新建一个ImportExportService.js文件，用于生成ImportExportProgress组件元素
javascript
import { createApp } from 'vue';
import ImportExportProgress from '@/components/ImportExportProgress.vue';

class ImportExportService {
    constructor() {
        this.vm = null;
    }

    start() {
        /** 传递给组件的props */
        const props = {
            title,
            content,
            cancelName,
            onClose: this.close.bind(this) // 防止this指向丢失
        };
        if (this.vm) {
            /** 如存在则无需重新生成实例 */
            return
        }
        const container = document.createElement('div');
        /** 方式一 */
        this.vm = createApp(ImportExportProgress, props); // 生成组件实例以及dom元素
        this.vm.mount(container); // 根据vnode渲染dom到容器
		/** 方式二 */
        // this.vm = createVNode(ImportExportProgress, props); // 生成组件实例
        // render(this.vm, container); // 根据vnode渲染dom到容器
        document.body.appendChild(container); // 将组件DOM元素添加到页面上
        return this.close.bind(this);
    }
    
    close() {
        this.vm.unmount(); // 卸载组件实例, 触发unMounted事件
        this.vm = null; // 清空内存实例, 进行垃圾回收
        document.body.removeChild(container); // 移除dom元素
    }

    end() {
        this.vm._instance.exposed.end(); // 调用组件内部方法
    }

    error({ title }) {
        this.vm._instance.exposed.error();
    }
}
export default new ImportExportService();

使用 ​

javascript
import ImportExportService from './ImportExportService.js';

const close = ImportExportService.start() // 生成组件

close() // 关闭组件

好处 ​

  1. 函数式调用，无需关心组件层面
  2. 动态生成，无需一直挂载组件
  3. 只有当使用的时候才占用内存，优化性能(一丢丢)

注意事项 ​

  1. 使用createApp生成vnode时，传入的props不是响应式的，如果需要响应式则需要传入ref/reactive代理的变量

    • 只能传入name整个对象，不可传入name.value，否则会丢失响应式，且在模板中使用需要.value，无法自动解包，只有在vue文件中声明的响应式变量才能够被编译器自动解包

      javascript
      const name = ref('张三')

const app = createApp('组件', { name })  // good
const app = createApp('组件', { name: name.value }) // bad，丢失响应式

// 组件内接收
const props = defineProps({
    name: { type: Object, default: () => ({}) } // good
    name: { type: String, default: "" } // bad
})
      html
      // 模板使用
<div>
    {{ name.value }} - good
    {{ name }} - bad
</div>
